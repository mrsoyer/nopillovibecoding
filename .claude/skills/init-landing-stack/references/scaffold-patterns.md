## Pre-requis

Verifier en debut de phase 2 :

```bash
# Node 22+
node --version | grep -E "^v(2[2-9]|[3-9][0-9])\." || echo "FATAL: Node 22+ requis"

# CLI Supabase
command -v supabase || npm install -g supabase

# CLI Netlify
command -v netlify || npm install -g netlify-cli

# gh CLI (optionnel, pour Q6 = github)
command -v gh && gh auth status 2>/dev/null || echo "WARN: gh non installe ou non auth, mode Q6=none"
```

Si Node < 22 : STOP avec instruction `brew install node@22` ou `nvm install 22`.

## Sequence Astro (MVP)

### Etape 2.1 — Dossier racine + git

```bash
# Demander a l'utilisateur ou creer (par defaut, dans le dossier courant)
PROJECT_DIR="${HOME}/projets/{{PROJECT_NAME}}"
# Ou : PROJECT_DIR="$(pwd)/{{PROJECT_NAME}}"

mkdir -p "$PROJECT_DIR"
cd "$PROJECT_DIR"
git init -b main
```

### Etape 2.2 — Init Astro dans `front/`

```bash
# Astro 6 + TypeScript strict, sans git interne (on a deja un git racine)
npm create astro@latest front -- \
  --template minimal \
  --typescript strict \
  --install \
  --git no \
  --yes
```

Effet : creation `front/package.json`, `front/astro.config.mjs`, `front/tsconfig.json`, `front/src/`.

### Etape 2.3 — Ajouter Tailwind 4 + Supabase JS

```bash
cd front
npx astro add tailwind --yes
npm install @supabase/supabase-js
cd ..
```

Note : `npx astro add tailwind` modifie `astro.config.mjs` automatiquement pour ajouter `import tailwind from '@astrojs/tailwind'`.

### Etape 2.4 — Structure annexe

```bash
mkdir -p supabase/{migrations,functions/contact-form}
mkdir -p .claude/{rules,skills}
mkdir -p docs
```

### Etape 2.5 — Generation des fichiers depuis templates

Pour chaque template, lire `assets/templates/...tmpl`, faire substitution `{{VAR}}`, ecrire dans le projet.

Templates a appliquer (MVP) :

| Source template | Destination dans le projet |
|-----------------|----------------------------|
| `astro/CLAUDE.md.tmpl` | `CLAUDE.md` |
| `astro/netlify.toml.tmpl` | `netlify.toml` |
| `astro/.gitignore.tmpl` | `.gitignore` |
| `astro/README.md.tmpl` | `README.md` |
| `astro/styles/global.css.tmpl` | `front/src/styles/global.css` |
| `astro/lib/supabase.ts.tmpl` | `front/src/lib/supabase.ts` |
| `astro/pages/index.astro.tmpl` | `front/src/pages/index.astro` |
| `astro/layouts/Base.astro.tmpl` | `front/src/layouts/Base.astro` |
| `astro/components/Hero.astro.tmpl` | `front/src/components/Hero.astro` |
| `astro/components/ContactForm.tsx.tmpl` | `front/src/components/ContactForm.tsx` |
| `supabase/config.toml.tmpl` | `supabase/config.toml` |
| `supabase/migrations/create_leads.sql.tmpl` | `supabase/migrations/{{TIMESTAMP}}_create_leads.sql` |
| `supabase/functions/contact-form/index.ts.tmpl` | `supabase/functions/contact-form/index.ts` |
| `claude-rules/frontend.md.tmpl` | `.claude/rules/frontend.md` |
| `claude-rules/backend.md.tmpl` | `.claude/rules/backend.md` |
| `claude-rules/claude-md.md.tmpl` | `.claude/rules/claude-md.md` |

Methode de substitution (bash) :

```bash
substitute() {
  local src="$1"
  local dst="$2"
  sed \
    -e "s|{{PROJECT_NAME}}|$PROJECT_NAME|g" \
    -e "s|{{PROJECT_TYPE}}|$PROJECT_TYPE|g" \
    -e "s|{{SB_URL}}|$SB_URL|g" \
    -e "s|{{SB_ANON_KEY}}|$SB_ANON_KEY|g" \
    -e "s|{{SB_PROJECT_REF}}|$SB_PROJECT_REF|g" \
    -e "s|{{NL_SITE_URL}}|$NL_SITE_URL|g" \
    -e "s|{{NL_SITE_NAME}}|$NL_SITE_NAME|g" \
    -e "s|{{TIMESTAMP}}|$(date +%Y%m%d%H%M%S)|g" \
    "$src" > "$dst"
}
```

Note : certaines vars (SB_URL, NL_SITE_URL) ne sont disponibles qu'apres phases 3-4. Le scaffold initial laisse des placeholders, ils sont remplaces apres deploy.

## Verification post-scaffold

```bash
# Doit retourner 0 pour tous
test -f front/package.json && echo "OK front/package.json"
test -f front/astro.config.mjs && echo "OK astro config"
test -f supabase/config.toml && echo "OK supabase config"
test -f supabase/functions/contact-form/index.ts && echo "OK edge fn"
test -f netlify.toml && echo "OK netlify.toml"
test -f CLAUDE.md && echo "OK CLAUDE.md"

# CLAUDE.md doit faire < 80 lignes
LINES=$(wc -l < CLAUDE.md)
[ "$LINES" -lt 80 ] && echo "OK CLAUDE.md $LINES lignes" || echo "FAIL: CLAUDE.md $LINES >= 80"

# package.json doit contenir les bonnes deps
cd front && npm ls @supabase/supabase-js astro @astrojs/tailwind --depth=0 2>&1 | grep -v UNMET
cd ..
```

Si une verification echoue : STOP, afficher quelle etape a foire, proposer le fix.

## Commandes Vite (V1.1 future)

Reserve pour la version V1.1 du skill. NE PAS implementer dans le MVP.

Reference : [docs/stack-landing-claude-code/03-setup-vite-netlify.md](../../../webflowlanding/docs/stack-landing-claude-code/03-setup-vite-netlify.md).

Commandes prevues (pour reference uniquement) :

```bash
npm create vite@latest front -- --template react-ts
cd front
npm install
npm install -D vite-react-ssg @netlify/vite-plugin tailwindcss @tailwindcss/vite
npm install @supabase/supabase-js
```

Build script a override dans `package.json` : `"build": "vite-react-ssg build"`.
