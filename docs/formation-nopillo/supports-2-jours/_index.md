# Supports formation 2 jours — Index

## Vue d'ensemble

Ce dossier contient **tous les supports operationnels** pour animer la formation Nopillo 2 jours definie dans `docs/formation-nopillo/05-format-2-jours.md` :

- Plans slide-by-slide (J1 + J2)
- Exercices avec consignes et criteres reussite
- Checklist animateur (Thomas)
- Kit participant (livrables a remettre)
- Evaluation skill check + NPS

## Promesse formation 2 jours

A la fin des 2 jours, l'equipe Nopillo a **industrialise** sa production landings :
- 5+ skills Claude Code reutilisables capitalises
- Workflow MCP rode (Webflow + HubSpot + Google + Meta)
- Tracking complet (GA4 + Pixel + CAPI + Consent V2)
- 2-3 landings produites en pratique

## Comment utiliser ces supports

### En tant que formateur (Thomas)

1. **J-14** : lire integralement `05-checklist-animateur.md` puis preparer les supports
2. **J-7** : finaliser slides (`01-slides-jour-1.md` + `02-slides-jour-2.md`) en outil Slidev / Keynote
3. **J-3** : envoyer skill check pre-formation (`07-evaluation-skill-check.md`)
4. **J0 matin** : checklist matin de `05-checklist-animateur.md`
5. **Pendant** : suivre les exercices de `03-exercices-jour-1.md` et `04-exercices-jour-2.md`
6. **J+1** : envoyer kit participant (`06-kit-participant.md`) + skill check post + NPS
7. **J+30 et J+90** : suivi Niveaux 3 et 4 Kirkpatrick

### En tant que participant Nopillo

Les supports `06-kit-participant.md` listent ce que vous recevez. Le reste du dossier est materiel formateur, pas a partager directement.

## Pedagogie de reference

Ces supports respectent les principes de `docs/pedagogie-formation/` :

- Ratio **20% theorie / 80% pratique** (cf `03-format-2-jours.md`)
- 5 techniques d'animation (cf `05-techniques-animation.md`) : pair programming, demo + reproduce, code review, energizers, 1-to-1
- Evaluation Kirkpatrick 4 niveaux (cf `06-evaluation.md`)
- Materiel complet (cf `07-materiel-pedagogique.md`)

## Methodologie de reference

L'ADN de la formation est la methode "Documentation-First" (`docs/methodologie-documentation-first/`) :

- Principe 1 : enqueter avant de coder (`/doc-maker`)
- Principe 2 : cadrer avant Webflow (`/cdc-maker`)
- Principe 3 : capitaliser apres la 2e fois (`/skill-maker`)
- Principe 4 : executer via MCP
- Principe 5 : verifier chaque output
