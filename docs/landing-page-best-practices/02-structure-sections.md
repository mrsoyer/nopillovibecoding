# Structure & Ordre des Sections

## Vue d'Ensemble

Une LP performante suit une sequence narrative qui repond aux objections progressivement. Les sections sont ordonnees selon la logique psychologique du visiteur : attention -> interet -> desir -> action (AIDA), couplee aux patterns de scan visuel **F-pattern** ou **Z-pattern**.

## Structure Type d'une Landing Page (10 Sections)

| # | Section | Role | Obligatoire ? |
|---|---------|------|---------------|
| 1 | **Hero** | Headline + sub + visuel + CTA primaire | OUI |
| 2 | **Trust bar** | Logos clients / media mentions | OUI (si dispo) |
| 3 | **Problem** | Nomme la douleur du visiteur | OUI (recommande) |
| 4 | **Solution / Value Prop** | Comment vous resolvez | OUI |
| 5 | **Features / Benefits** | Bullets outcomes-driven | OUI |
| 6 | **Social Proof** | Testimonials + stats + reviews | OUI |
| 7 | **Demo / How It Works** | Video ou screenshots produit | Recommande |
| 8 | **Pricing / Offer** | Prix + ce qu'on obtient | OUI si transactionnel |
| 9 | **FAQ** | Leve les objections finales | Recommande |
| 10 | **Final CTA + risk reversal** | Garantie + dernier push | OUI |

## Ordre Narratif Detaille

### Sequence Problem -> Story -> Results -> Action (LearnWorlds)

1. **Hook attentionnel** (hero) — promesse claire en 3-5 sec
2. **Probleme reconnu** — "Tu galeres avec X ?"
3. **Solution presentee** — "Voici comment on resout X"
4. **Preuve concrete** — "Voici ce que les clients ont obtenu"
5. **Action invitee** — "Demarre maintenant"

### Sequence Hero -> Problem -> Solution -> Proof -> Offer (Unbounce)

```
HERO            (headline + visual + CTA)
PROBLEM         (pain point clair)
SOLUTION        (votre offre comme reponse evidente)
SOCIAL PROOF    (testimonials, logos, case studies)
THE OFFER       (ce qu'ils obtiennent en disant oui)
```

## Patterns de Scan Visuel

### Z-Pattern
Pour LP **courte avec peu de texte** (ex: SaaS feature page).

```
[LOGO]            [NAV/CTA]
            \
              \
                \
[VISUAL]         [HEADLINE + CTA]
```

Eye flow : haut-gauche -> haut-droit -> bas-gauche -> bas-droit. Met le CTA dans le dernier point d'attention.

### F-Pattern
Pour LP **longue avec beaucoup de texte** (ex: course, B2B).

```
[==========================] (ligne 1 : scan complet)
[==========]                  (ligne 2 : scan partiel)
[====]                        (vertical scan a gauche)
[====]
[====]
```

Place les infos cles en debut de ligne et a gauche. Sous-titres percutants pour reaccrocher l'attention.

## Wireframe — Framework CopyHackers (8 Etapes)

1. **Resolution standard** : commencer en 1920x1080
2. **Design minimal** : noir, blanc, gris uniquement
3. **Image placeholders** : rectangles X-marked + indication contenu
4. **Typographie consistante** : H1 / H2 / body uniformes
5. **Lecture confortable** : pas de full-width sur les blocs de texte
6. **Edition ruthless** : trimmer headlines / body trop longs
7. **White space strategique** : aerer pour respirer
8. **Visual blocking** : separer sections par lignes / shading gris clair

**Benefice cle** : Le wireframe avec copy fait shifter le feedback de "Est-ce lisible ?" a "**Est-ce notre message ?**".

## Methode de Construction (5 Etapes)

1. **Lister les sections** que vous voulez couvrir
2. **Ranger par importance** pour le client (pas pour vous)
3. **Ecrire headers + subheaders** AVANT le copy detaille
4. **Setup en F ou Z pattern** selon longueur
5. **Finaliser le headline** au moment du wireframe (definit le theme)

## Contraintes de Longueur

| Type LP | Longueur recommandee |
|---------|---------------------|
| LP simple (lead gen) | < 500 mots, 1 ecran scrolle |
| LP SaaS feature | 800-1500 mots, 3-5 sections |
| LP formation / course | 2000-4000 mots, 8-12 sections |
| LP sales (long-form) | 3000-8000 mots, 12-20 sections |

> Unbounce : "Les LP < 350 mots performent mieux en media/entertainment. Les pages longues fonctionnent en B2B/courses si structurees en accordeons FAQ."

## Anti-Patterns Structurels

| Anti-pattern | Correction |
|--------------|-----------|
| Info critique below-the-fold | Headline + value prop + CTA above-the-fold |
| Sections sans hierarchie | Ordre narratif Problem -> Solution -> Proof |
| Une LP = 3 objectifs | 1 LP = 1 objectif (sinon -266% conversion) |
| Naviguer comme un site classique | Supprimer la nav (ou minimaliste) |
| Cramer le visual avec le headline | Hierarchie : headline > CTA > visual |
